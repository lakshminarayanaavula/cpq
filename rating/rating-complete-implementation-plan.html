This implementation plan combines the refactoring of the existing RatingServiceImpl into a configurable MFlow pipeline with the new features defined in the PRD including Event Ingestion, Discounting Engine, Credits Engine, Tax Engine, and Re-rating capabilities.

Persists rated transactions and generates bills. Will integrate with new charge model.

API Specifications

Event Ingestion APIs

POST /v1/events Ingest single usage event
POST /v1/events/batch Batch event ingestion (async)
GET /v1/events Query events with filters

Rating APIs

POST /v1/rate Real-time rating (<200ms)
POST /v1/rate/bulk Bulk rating with job tracking
GET /v1/charges Query charge lines
GET /v1/charges/{id}/explain Charge breakdown explanation

Credits APIs

POST /v1/credits/apply Add credits to account
GET /v1/credits/balance Query credit balance

Invoice APIs

POST /v1/invoices/generate Generate invoice for period
GET /v1/invoices/preview Preview invoice without commit

Re-rating APIs

POST /v1/rerate Re-rate historical events
