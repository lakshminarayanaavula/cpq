1. Executive Overview

This document outlines a comprehensive plan to refactor the monolithic RatingServiceImpl.process() method into a configurable, step-based MFlow pipeline using the existing NativeMFunction pattern and JSON configuration system.

Goal: Transform a 166-line monolithic method into 12 independent, testable, and configurable steps.

Key Benefits

Benefit Description
Configurable Enable/disable steps via JSON without code changes
Extensible Add new steps by creating new NativeMFunction implementations
Testable Each function can be unit tested in isolation
Observable Step-level execution tracking and timing
Tenant-Specific Different tenants can have custom rating flows
Backward Compatible Feature flag allows gradual rollout

1.2 Create Rating Step Base Interface

backend/base/src/main/java/com/monetize360/mpillotv2/modules/billing/rating/functions/RatingStepFunction.java
package com.monetize360.mpillotv2.modules.billing.rating.functions;

import com.fasterxml.jackson.databind.*;
import com.monetize360.mpillotv2.common.model.mfunction.*;
import com.monetize360.mpillotv2.modules.billing.rating.context.RatingContext;
import com.monetize360.mpillotv2.modules.mflow.service.MFlowExecutionContext;
import lombok.extern.slf4j.Slf4j;
import org.apache.commons.lang3.time.StopWatch;
import java.util.concurrent.TimeUnit;

@Slf4j
public abstract class RatingStepFunction implements NativeMFunction {

    protected abstract ObjectMapper getObjectMapper();
    protected abstract String getStepName();
    protected abstract void executeStep(RatingContext context) throws Exception;

    @Override
    public MFunctionOutput<JsonNode> execute(JsonNode input, MFlowExecutionContext context)
            throws Exception {
        StopWatch stopWatch = StopWatch.createStarted();
        String stepName = getStepName();

        log.info("Starting step: {}", stepName);

        try {
            // Parse input to RatingContext
            RatingContext ratingContext = getObjectMapper()
                .treeToValue(input, RatingContext.class);

            // Execute the step
            executeStep(ratingContext);

            // Record execution
            long elapsed = stopWatch.getTime(TimeUnit.MILLISECONDS);
            ratingContext.getExecutedSteps().add(stepName);
            ratingContext.getStepTimings().put(stepName, elapsed);

            log.info("Completed step: {} in {}ms", stepName, elapsed);

            return completed(getObjectMapper().valueToTree(ratingContext));

        } catch (Exception e) {
            log.error("Step {} failed: {}", stepName, e.getMessage(), e);
            throw e;
        }
    }
}

Each step is implemented as a Spring component extending RatingStepFunction.

  • Queries contract tables

  • Gets rate card from contract or service

  • Builds RateComputationInput
  • Executes ServiceRateComputationUtil

  • Resolves billing accounts

Example: ContractResolutionFunc

@Component("ratingContractResolutionFunc")
@RequiredArgsConstructor
@Slf4j
public class ContractResolutionFunc extends RatingStepFunction {

    private final RatingObjectsManager ratingObjectsManager;
    private final ObjectMapper objectMapper;

    @Override
    protected ObjectMapper getObjectMapper() { return objectMapper; }

    @Override
    protected String getStepName() { return "CONTRACT_RESOLUTION"; }

    @Override
    protected void executeStep(RatingContext context) throws Exception {
        RatingInput ratingInput = context.getRatingInput();
        UUID serviceId = context.getServiceDTO().getService().getId();
        String masterAccountId = ratingInput.getMasterAccountId();
        LocalDate transactionDate = ratingInput.getTransactionDate();
        String serviceInstanceId = ratingInput.getServiceInstanceId();

        // Query contract
        ContractRateCardDTO contractRateCardDTO = ratingObjectsManager.getContractRateCard(
            serviceId, masterAccountId, transactionDate, serviceInstanceId);

        if (contractRateCardDTO != null) {
            context.setContractResolved(true);
            context.setContractId(contractRateCardDTO.getContractId());
            context.setContractServiceId(contractRateCardDTO.getContractServiceId());
            context.setContractRateCardDTO(contractRateCardDTO);
            context.setBillingAccountShares(contractRateCardDTO.getDimNameToBillingAccInfo());
            context.setServiceStartDate(contractRateCardDTO.getServiceStartDate());
            context.setServiceEndDate(contractRateCardDTO.getServiceEndDate());

            if (MapUtils.isNotEmpty(contractRateCardDTO.getContractInputDimVals())) {
                context.setContractInputDimensions(contractRateCardDTO.getContractInputDimVals());
            }

            log.info("Contract resolved: contractId={}", contractRateCardDTO.getContractId());
        } else {
            context.setContractResolved(false);
            log.info("No contract found for serviceId={}", serviceId);
        }
    }
}

7. Phase 4: Flow Executor Service

backend/base/src/main/java/com/monetize360/mpillotv2/modules/billing/rating/RatingFlowService.java
@Service
@RequiredArgsConstructor
@Slf4j
public class RatingFlowService {

    private static final UUID RATING_FLOW_ID =
        UUID.fromString("f2000001-0000-0000-0000-000000000001");

    private final MFlowExecutor mFlowExecutor;
    private final ObjectMapper objectMapper;

    /**
     * Execute rating using MFlow pipeline
     */
    public List<TransactionFullDTO> process(RatingInput ratingInput) {
        log.info("Starting rating flow for serviceId: {}",
            ratingInput.getServiceId());

        StopWatch stopWatch = StopWatch.createStarted();

        try {
            // Build MAction
            MAction action = MAction.builder()
                .flowId(RATING_FLOW_ID)
                .payload(objectMapper.valueToTree(ratingInput))
                .build();

            // Execute flow
            JsonNode result = mFlowExecutor.execute(action);

            // Extract context from result
            RatingContext ratingContext = objectMapper
                .treeToValue(result, RatingContext.class);

            long elapsed = stopWatch.getTime(TimeUnit.MILLISECONDS);

            log.info("Rating flow completed in {}ms. Steps: {}",
                elapsed, ratingContext.getExecutedSteps());

            return ratingContext.getTransactionDTOs();

        } catch (Exception e) {
            log.error("Rating flow failed", e);
            throw new RuntimeException("Rating process failed", e);
        }
    }

9. Phase 6: Testing Strategy

Unit Test Example

@ExtendWith(MockitoExtension.class)
class InputValidationFuncTest {

    @Spy
    private ObjectMapper objectMapper = new ObjectMapper();

    @InjectMocks
    private InputValidationFunc inputValidationFunc;

    @Test
    void execute_withValidInput_shouldCreateContext() throws Exception {
        RatingInput input = new RatingInput();
        input.setServiceId(UUID.randomUUID());
        input.setMasterAccountId(UUID.randomUUID().toString());

        JsonNode inputNode = objectMapper.valueToTree(input);

        MFunctionOutput<JsonNode> result = inputValidationFunc.execute(inputNode, null);

        assertThat(result.getExecutionStatus().name()).isEqualTo("COMPLETED");

        RatingContext context = objectMapper
            .treeToValue(result.getOutput(), RatingContext.class);
        assertThat(context.getExecutedSteps()).contains("INPUT_VALIDATION");
    }

    @Test
    void execute_withMissingServiceId_shouldThrowException() {
        RatingInput input = new RatingInput();
        input.setMasterAccountId(UUID.randomUUID().toString());
        // serviceId is null

        JsonNode inputNode = objectMapper.valueToTree(input);

        assertThatThrownBy(() -> inputValidationFunc.execute(inputNode, null))
            .isInstanceOf(Exception.class);
    }
}

10. Rollout Strategy

Phase 1: Deploy with Flag Off

Deploy code with rating.use-flow-engine=false. Zero risk - no behavior change.

Phase 2: Internal Testing

Enable for internal/dev environment. Validate all step executions.

Phase 3: Staging Validation

Enable for staging with sample data. Compare results with legacy engine.

Phase 4: Limited Production

Enable for specific tenants or low-traffic accounts. Monitor closely.

Phase 5: Full Production

Enable for all production traffic. Continue monitoring.

Phase 6: Legacy Removal

After validation period, remove legacy implementation.

Risk Assessment

Phase Risk Level Rollback Strategy
Deploy with flag off None N/A
Internal testing Low Set flag to false
Staging Medium Set flag to false
Limited production Medium Set flag to false per-tenant
Full production Low Set flag to false globally
Legacy removal None Code already validated
